# Multi-view cropping 1.0 使用指南

## 🚀 快速启动

### 方法1: 简易启动 (推荐)
```bash
python run.py
```

### 方法2: 完整启动器 (带环境检查)
```bash
python start.py
```

### 方法3: 直接启动主程序
```bash
python main.py
```

## 📋 启动前检查

### 1. Python版本
确保使用 Python 3.8 或更高版本：
```bash
python --version
```

### 2. 依赖安装
如果出现模块缺失错误，运行：
```bash
pip install -r requirements.txt
```

### 3. 测试功能
运行功能测试确认一切正常：
```bash
# 测试核心算法
python test_core.py

# 测试GUI组件  
python test_gui.py
```

## 🎯 基本使用流程

### 步骤1: 添加图片
- 点击"添加文件"按钮选择图片
- 或直接拖拽图片到窗口
- 支持格式: PNG, JPG, JPEG, WEBP, BMP, TIFF

### 步骤2: 调整参数
- **融合度**: 控制相邻区域合并程度 (1-100)
- **过滤阈值**: 剔除太小的杂点区域 (100-50000)
- **移除水平线**: 去除图像中的水平干扰线
- **移除垂直线**: 去除图像中的垂直干扰线

### 步骤3: 查看结果
- 左侧显示原图和处理结果对比
- 绿色框框标出检测到的区域
- 右侧显示区域数量和图像信息

### 步骤4: 导出结果
- 点击"导出结果"按钮
- 选择输出目录
- 程序会为每张图片创建文件夹，保存分割后的各个区域

## 🎛️ 参数调优技巧

### 融合度过低
- **问题**: 一个完整对象被分割成多个小块
- **解决**: 增大融合度值 (如从15调到30)

### 过滤阈值过低  
- **问题**: 检测出太多小杂点
- **解决**: 增大过滤阈值 (如从2000调到5000)

### 水平线干扰
- **问题**: 表格线条等影响检测结果
- **解决**: 勾选"移除水平线"

### 垂直线干扰
- **问题**: 图像边框等影响检测结果  
- **解决**: 勾选"移除垂直线"

## 📁 输出结果说明

导出后的文件夹结构：
```
输出目录/
├── 图片1名/
│   ├── 图片1名_1.png
│   ├── 图片1名_2.png
│   └── ...
├── 图片2名/
│   ├── 图片2名_1.png
│   └── ...
└── ...
```

- 每个原图在一个独立的子文件夹中
- 文件名格式: `{原名}_{序号}.png`
- 按Z字顺序编号 (从上到下，从左到右)

## 🔧 高级功能

### 批量处理
- 可以同时添加多张图片进行处理
- 每张图片可以单独调整参数
- 参数设置会自动保存到该图片

### 切换图片
- 在文件列表中点击不同图片
- 参数会自动调整到该图片的设置
- 可以逐个优化每张图片的效果

### 实时预览
- 调整参数即时显示效果
- 无需等待处理完成
- 右下角显示检测到的区域数量

## ⚡ 性能优化建议

### 大图片处理
- 原始图片建议不要超过8K分辨率
- 大图片会自动缩放到预览尺寸处理，但导出使用原图

### 内存管理
- 同时处理图片建议不超过20张
- 如果内存不足，可以分批处理

### 处理速度
- 大部分1080p图片处理时间 < 500ms
- 4K图片处理时间约1-2秒

## 🐛 常见问题解决

### Q: 程序无法启动
A: 
1. 检查Python版本: `python --version`
2. 安装依赖: `pip install -r requirements.txt`
3. 运行测试: `python test_core.py`

### Q: 中文路径乱码
A: 程序已完全支持中文路径，如果仍有问题请确保使用UTF-8编码

### Q: 处理效果不理想
A: 
1. 先用默认参数试试
2. 逐步调整融合度和过滤阈值
3. 尝试启用线段过滤选项

### Q: 导出的图片质量差
A: 导出保持原始质量，如有损失可能是原始图片本身的问题

### Q: 程序卡死或崩溃
A:
1. 检查是否有损坏的图片文件
2. 减少同时处理的图片数量
3. 重启程序重新处理

## 📱 界面说明

- 底部显示当前操作状态
- 处理进度和结果提示
